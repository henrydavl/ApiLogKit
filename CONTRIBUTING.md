# Contributing to ApiLogKit

Bug reports, feature requests, and pull requests are welcome.

## Reporting bugs

Open an issue and include:
- iOS version and device/simulator
- Xcode and Swift version
- A minimal reproduction or description of the unexpected behavior

## Suggesting features

Open an issue describing what you'd like and why it's useful. Check existing issues first to avoid duplicates.

## Submitting a pull request

1. Fork the repo and create a branch from `main`
2. Make your changes — keep them focused on a single concern
3. Verify the package builds cleanly in Xcode (`⌘B` on the package target)
4. Open a PR with a clear description of what changed and why

## Code style

- Follow the conventions already present in the codebase
- No third-party dependencies — the library has none and should stay that way
- iOS 15+ compatibility must be maintained
